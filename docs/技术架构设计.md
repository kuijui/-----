### 1.2 技术栈选型

#### 前端（微信小程序）
- **框架**：微信小程序原生开发
- **UI组件库**：Vant Weapp（可选）
- **状态管理**：全局状态 + 本地存储
- **网络请求**：wx.request 封装

**为什么选择原生开发？**
- ✅ 性能最优，体验流畅
- ✅ 官方支持完善
- ✅ 包体积小
- ✅ 学习成本低

#### 后端服务
- **语言**：Node.js 18+
- **框架**：Express.js / Koa2
- **ORM**：Sequelize / Prisma
- **API文档**：Swagger

**为什么选择Node.js？**
- ✅ 前后端统一语言，开发效率高
- ✅ 异步IO性能好
- ✅ npm生态丰富
- ✅ 适合快速迭代

#### 数据库
- **关系型数据库**：MySQL 8.0
  - 用户数据、订单数据、会员数据
- **缓存数据库**：Redis 7.0
  - 用户token、限流计数、热点数据

#### 云服务
- **服务器**：腾讯云 / 阿里云
- **对象存储**：腾讯云COS / 阿里云OSS
- **CDN**：腾讯云CDN
- **监控**：腾讯云监控 / Sentry

#### AI服务
- **主要选择**：
  - OpenAI GPT-4 / GPT-3.5-turbo
  - 国内：通义千问 / 文心一言 / 智谱AI
- **备选方案**：
  - 多个API轮询，降低成本
  - 自建模型（成本高，不推荐初期）

## 3. 核心流程设计

### 3.1 用户登录流程

```
用户点击登录
    ↓
wx.login() 获取code
    ↓
发送code到后端
    ↓
后端调用微信API换取openid
    ↓
查询用户是否存在
    ↓
不存在 → 创建新用户 → 初始化免费次数
    ↓
生成JWT token
    ↓
返回token + 用户信息
    ↓
前端存储token
```

**接口设计**：
```javascript
POST /api/auth/login
Request: { code: "wx_code" }
Response: {
  token: "jwt_token",
  userInfo: {
    id: 1,
    nickname: "用户昵称",
    avatar: "头像URL",
    memberType: "free",
    remainingCount: 3
  }
}
```

### 3.2 文案生成流程

```
用户填写表单
    ↓
前端校验（主题不为空）
    ↓
检查剩余次数
    ↓
次数不足 → 提示开通会员
    ↓
显示loading
    ↓
调用后端API
    ↓
后端验证token
    ↓
检查用户权限和次数
    ↓
构建AI prompt
    ↓
调用AI API生成文案
    ↓
内容审核（敏感词过滤）
    ↓
扣除使用次数
    ↓
保存生成记录
    ↓
返回生成结果
    ↓
前端展示结果
```

**接口设计**：
```javascript
POST /api/content/generate
Request: {
  topic: "春季穿搭",
  description: "分享几套适合春天的穿搭",
  style: "种草推荐",
  length: "medium"
}
Response: {
  id: 123,
  titles: [
    "🌸春天这样穿，回头率200%！",
    "姐妹们！这几套春季穿搭绝了！",
    "春日氛围感穿搭｜温柔又高级"
  ],
  content: "生成的正文内容...",
  tags: ["#春季穿搭", "#OOTD", "#穿搭灵感"],
  score: 85,
  remainingCount: 2
}
```

### 3.3 支付流程

```
用户选择套餐
    ↓
点击支付
    ↓
前端调用后端创建订单
    ↓
后端生成订单号
    ↓
调用微信统一下单API
    ↓
返回支付参数
    ↓
前端调用wx.requestPayment
    ↓
用户完成支付
    ↓
微信回调后端
    ↓
验证支付结果
    ↓
更新订单状态
    ↓
开通会员权益
    ↓
发送模板消息通知
```

**接口设计**：
```javascript
POST /api/payment/create-order
Request: {
  productType: "monthly_member",
  amount: 19.9

  orderId: "ORDER123456",
  paymentParams: {

  }
}
```

### 5.2 AI调用优化

**成本控制**：
- 使用GPT-3.5-turbo而非GPT-4（成本降低10倍）
- 限制max_tokens，避免过长输出
- 使用缓存，相同输入直接返回

**质量保证**：
- temperature设置为0.7-0.9（创意性）
- 多次生成取最优
- 人工审核样本，优化prompt

**性能优化**：
- 异步调用，不阻塞主线程
- 超时设置（10秒）
- 失败重试机制（最多3次）

## 6. 安全设计

### 6.1 认证鉴权

**JWT Token**：
```javascript
{
  userId: 123,
  openid: "wx_openid",
  exp: 1234567890  // 7天过期
}
```

**权限验证**：
- 每个API请求验证token
- 检查用户状态（是否被封禁）
- 检查会员权限

### 6.2 防刷机制

**限流策略**：
- 免费用户：每日3次
- 月度会员：每日30次
- 单个IP：每分钟最多10次请求
- 单个用户：每秒最多1次生成请求

**实现方式**：
```javascript
// Redis计数器
const key = `rate_limit:${userId}:${date}`;
const count = await redis.incr(key);
if (count === 1) {
  await redis.expire(key, 86400); // 24小时过期
}
if (count > limit) {
  throw new Error('超出每日限制');
}
```

### 6.3 内容安全

**敏感词过滤**：
- 接入腾讯云内容安全API
- 本地敏感词库（政治、色情、暴力）
- 违规内容拒绝返回

**数据加密**：
- 用户敏感信息加密存储
- HTTPS传输
- 支付信息不存储

## 7. 性能优化

### 7.1 前端优化

- **分包加载**：按页面分包，减少首屏加载
- **图片优化**：使用webp格式，CDN加速
- **请求优化**：合并请求，减少网络开销
- **缓存策略**：历史记录本地缓存

### 7.2 后端优化

- **数据库优化**：
  - 索引优化（user_id, created_at）
  - 读写分离
  - 连接池管理
  
- **缓存策略**：
  - 用户信息缓存（1小时）
  - 热门主题缓存（6小时）
  - 会员状态缓存（30分钟）

- **异步处理**：
  - 日志异步写入
  - 统计数据异步计算

### 7.3 AI调用优化

- **智能缓存**：相似主题复用结果
- **批量处理**：合并多个请求
- **降级策略**：AI服务故障时返回模板

## 11. 技术选型对比

### 11.1 小程序框架对比

| 框架 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| 原生 | 性能最优、官方支持 | 开发效率略低 | ⭐⭐⭐⭐⭐ |

| Nest.js | TypeScript、企业级 | 学习成本高 | ⭐⭐⭐ |

**结论**：选择Express（快速开发）
